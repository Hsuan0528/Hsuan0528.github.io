1.  **檔名規範**：GitHub 對大小寫非常敏感！如果你的檔案叫 `Photo.JPG`，你在 `content.txt` 裡寫 `photo.jpg`，**圖片會顯示不出來**。建議所有圖片檔名都使用「小寫英文 + 連字號」 (例如：`my-vacation-photo.jpg`)。
2.  **檔案大小**：GitHub 對於單一檔案的大小有限制（建議不要超過 50MB）。如果是超大解析度的照片，建議先用電腦稍微壓縮一下再上傳，這樣讀者打開網頁時才不會轉圈圈轉很久。
3.  **上傳後的檢查**：
    *   推送到 GitHub 後，請檢查你的 GitHub 倉庫頁面，確認圖片檔案確實出現在對應的資料夾內。
    *   如果網頁上沒看到圖片，按 `F12` 打開開發者工具，切換到 **Console (主控台)**，通常會看到一個 `404 (Not Found)` 的紅字，那個紅字會明確告訴你網頁在找哪個路徑的圖片找不到，你就知道哪裡路徑錯了。
